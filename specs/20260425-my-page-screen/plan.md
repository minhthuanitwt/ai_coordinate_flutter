**Branch**: `20260425-my-page-screen` | **Date**: 2026-04-25 | **Spec**: `/Users/dongdm/Develop/Source/doi/new/ai_coordinate_flutter/specs/20260425-my-page-screen/spec.md`

## Summary

Implement a real authenticated My Page tab in Flutter by replacing the current
placeholder `my_profile` screen with a structured feature module (`page`,
`state`, `view_model`) that mirrors the web My Page information architecture:
profile header, user stats, credit balance summary, and generated-image
gallery. Keep existing auth-gate and redirect behavior from shell navigation.

## Technical Context

**Language/Version**: Dart 3.10.x, Flutter 3.38.9  
**Primary Dependencies**: auto_route, hooks_riverpod, freezed, supabase_flutter, slang/slang_flutter  
**Storage**: Supabase (Postgres tables + RPC + Storage URLs) via existing service layer  
**Testing**: `flutter_test` (unit/widget where practical), `dart analyze`, targeted manual auth-gate verification  
**Target Platform**: Flutter mobile app (Android/iOS first), desktop/web compatibility should not regress  
**Project Type**: Single Flutter mobile app  
**Performance Goals**: First meaningful My Page content rendered in one pass with progressive section loading (no full-screen reload after initial frame)  
**Constraints**: Preserve current app shell + router + Riverpod patterns; explicit loading/error/empty/success states; generated files only via toolchain  
**Scale/Scope**: One protected tab (`my-page`) and its data integration; sub-pages (`account`, `contact`, `credits`) are out of this slice
